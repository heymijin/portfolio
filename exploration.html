Exploration · 탐색의 기록

Things I tried —
and what I found out

성공한 것만 올려두는 포트폴리오는 절반의 이야기다. 여기에는 내가 시도했고, 끝까지 가지 못했거나, 방향이 아니라는 걸 확인한 것들을 기록한다. 실패가 아니라 탐색이었다.

2023.03 – 2023.08

Data Engineering

방향 확인

플레이데이터 부트캠프 — 뒷단이 생각보다 훨씬 깊었다

Spark · Airflow · Hadoop · Hive · SQL · Python · Tableau · PySpark

Siemens에서 KNIME으로 데이터를 자동화하면서 더 깊이 배워보고 싶었다. 데이터를 구조화하고 자동화하는 엔지니어링을 직접 익혀보고 싶어서 국비 부트캠프를 시작했다.

막상 들어가보니 내가 관심 있던 것보다 훨씬 뒷단이었다 — 분산처리, 파이프라인 아키텍처, Hadoop 생태계. 제대로 따라가기 어려웠다. 그래도 데이터 분석 기초와 Python, SQL을 익혔고, Spotify 스트리밍 히스토리 분석과 Seoul City Bike 데이터 파이프라인 프로젝트를 완성했다.

결론: 데이터 엔지니어링은 내 방향이 아니었다. 하지만 이 탐색이 없었다면 지금처럼 IDE, GitHub, 코드 기반 툴을 편하게 다루지 못했을 것이다. 소프트웨어 시스템이 어떻게 작동하는지에 대한 감각을 처음 얻은 시간이었다.

2025.12 – 2026.03

UX Design

수료, 미완성

Google UX Certificate — 수료했지만 포트폴리오를 완성하지 못했다

Figma · Framer · UX Research · Wireframing · Prototyping

디자인에 대한 관심은 오래됐다. 직접 만들어보고 싶다는 생각이 있었고, Google UX Design Certificate를 수료했다. Figma로 프로토타이핑을 익히고 세금 신고 앱, NYC 지출 대시보드, 멤버십 가입 플로우 등을 케이스스터디로 설계했다.

수료증은 땄다. 하지만 UX 포트폴리오를 끝까지 완성하지 못했다. 만드는 과정에서 계속 멈췄다 — 모호한 크리에이티브 판단보다 명확한 실행과 구조가 있는 환경에서 내가 더 잘 작동한다는 걸 서서히 확인했다.

결론: UX 디자이너로 가는 길은 아니었다. 하지만 이 탐색이 지금 Ops 방향을 선택한 근거가 됐다. 무엇이 맞는지를 알기 위해 무엇이 안 맞는지를 먼저 확인해야 했다.

2025

Side Project

작동, 단기 운영

Cleo 챗봇 — 처음으로 아이디어를 작동하는 것으로 만든 경험

Python · Gemini API · Telegram API · PythonAnywhere · Cursor

영어 실력을 유지하고 싶었고, 매일 시간을 트래킹하고 싶었다. 두 가지를 동시에 해결할 수 있는 Telegram 챗봇을 만들기로 했다. AI의 도움을 받아 Gemini API와 Telegram API를 연결했고, PythonAnywhere에 배포해서 실제로 72시간 운영해봤다.

코드를 직접 처음부터 짠 게 아니라, AI가 알려주는 방향대로 Cursor에서 조립하는 방식이었다. 완성도 있는 결과물이라고 말하기는 어렵다.

결론: "코드를 잘 못 써도 작동하는 걸 만들 수 있다"는 걸 처음 경험했다. 아이디어에서 실행까지 혼자 끝냈다는 것 자체가 의미 있었고, 이후 도구를 두려워하지 않게 됐다.
